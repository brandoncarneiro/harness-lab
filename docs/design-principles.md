# Design Principles

## Prompts are not enough

Prompts are instructions. Harnesses are systems.

## Every stage produces an artifact

If a stage does not produce something inspectable, it is probably not a real stage.

## Workflows beat roleplay

Do not create fake job titles for agents unless the role has a clear input, output, and success criteria.

## Human approval matters

Risky actions should stop and ask.

## Start manual, then automate

A workflow should work as a prompt harness before becoming a production system.

## Keep it inspectable

Users should see prompts, outputs, assumptions, diffs, audits, and decisions.

## Improve the harness, not just the output

A failed run should create a better harness.
